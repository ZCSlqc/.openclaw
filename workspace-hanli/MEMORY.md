# 辩论任务执行规则（规范版）

## 一、发起辩论规则

### 1. 智能体选择

开启辩论任务时，默认不新建临时智能体，直接与已有辩手开展通信。

### 2. 目标辩手与 SessionKey

当前唯一对方辩手为「中禅寺」，对应固定 `SessionKey`，禁止修改、禁止使用 `label` 查找：

`sessionKey="agent:zcs:feishu:feishu_zcs:direct:ou_57fc20d82d39a490d881bfe6db969b77"`

### 3. 通信方式

优先通过 `sessions_send` 工具，将辩题、立场及辩论规则发送给目标辩手；仅当目标智能体不存在，或用户明确要求新开会话时，才使用 `sessions_spawn` 工具。

### 4. 辩题发送模板

调用 `sessions_send` 工具时，在 tool_call 基础上采用以下模板，替换 {} 内占位内容即可：

```plain text
sessions_send(
  sessionKey={对方辩手 sessionKey},
  message="
【辩题】：{填写具体辩题}
【立场】：{填写己方/对方立场，如正方/反方 + 具体观点}
【辩论规则】：
1. 发言规范：围绕辩题展开，逻辑清晰、论据充分，可联网搜索依据，禁止偏离主题、人身攻击及不文明表述。
2. 同步要求：所有辩论内容（立论、驳论、回应等）需完整同步至指定群聊，不得私下辩论。
3. 同步模板：调用 `message` 工具时，在 tool_call 基础上采用以下模板，替换<>内占位内容即可：
message(
  action="send",
  message="<填写完整辩论发言内容>",
  target="chat:oc_948d9a67d6185924de6f1d74b8382935",
  channel="feishu"
)
4. 通用约束：严格使用指定 target，不得修改；工具调用需遵循模板，避免参数缺失、格式错误。
5. 职责要求：双方需及时回复对方辩手，高效推进辩论。
"
)
```
